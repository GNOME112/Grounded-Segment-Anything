# Contributing to Grounded Segment Anything

Thank you for considering contributing to the Grounded Segment Anything project! We welcome contributions to improve the project and are excited to see what you bring to the table.

## Contribution Guidelines
- **Fork the repository**: Start by forking the repository to your own GitHub account.
- **Clone the repository**: Clone the forked repository to your local machine.
- **Make your changes**: Create a new branch for your feature or bug fix. Make your changes in this branch.
- **Keep your commits clean**: Write clear and concise commit messages. 

## Pull Request Procedures
1. After making your changes, push them to your forked repository.
2. Go to the original repository and create a pull request.
3. Make sure to reference any related issues in your pull request description.
4. Wait for feedback from the maintainers. They can request changes or approve your request.

## Code Standards
- Follow [the style guide](link to style guide) for writing your code.
- Ensure that your code passes all existing tests and include new tests for any new features.
- Document your code appropriately, following the project’s documentation standards.

We look forward to your contributions! Feel free to reach out if you have any questions.
